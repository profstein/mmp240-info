Introduction to Responsive Web Design

Introduction to Responsive Web Design

The idea of responsive web design (RWD) was introduced by Ethan Marcotte in 2010 in his article “Responsive Web Design” on A List Apart. He proposed thinking about layouts that adapt gracefully to different screen sizes.

Before RWD, most websites were fixed-width and optimized for a desktop monitor (~960px). When smartphones launched, users had to zoom and scroll just to read a line of text. Designers often created separate mobile versions, which doubled maintenance and fractured the experience.

Instead, Marcotte suggested building one flexible system that responds to the viewport. The same site adjusts itself based on the device.

The Three Core Principles of RWD

We have already covered the first two principles, and we will learn the third in Project 3.

Fluid Layouts

Layout containers should use flexible units instead of fixed pixels.

  • Use fractional (fr) units in CSS Grid or percentages in other contexts.
  • This keeps layout elements proportional as the viewport width changes.

Example using fr units in grid:

.container {
  display: grid;
  grid-template-columns: 1fr 2fr;
}

Example using percentages (centered with margins):

.container {
  width: 85%;
  margin: 0 auto;
}

Flexible Media

Images and videos must resize within their containers.

  • Use the base responsive media rule for img and video elements.
img, video {
  max-width: 100%;
  height: auto;
}

Responsive Image Example (CodePen): https://codepen.io/profstein/pen/mqMBoV

  • For art direction, use the <picture> element or srcset to serve different crops or resolutions.

Responsive Picture Element - Art Direction Example (CodePen): https://codepen.io/profstein/pen/JoGLoKX

  • For embedded video, wrap the player in a wrapper that preserves aspect ratio.

Responsive Video Embed Example (CodePen): https://codepen.io/profstein/pen/mrMLRj

Media Queries

We will learn media queries for Project 3.

  • Media queries apply different CSS rules based on device or viewport features.
  • This class uses a mobile-first approach: start with small-screen styles, then layer on @media (min-width: ...).

Example:

main {
  display: grid;
  grid-template-columns: 1fr;
}

@media (min-width: 600px) {
  main {
    grid-template-columns: 2fr 1fr;
  }
}

Media Queries in action on CodePen: https://codepen.io/profstein/pen/wBKVvNE

Summary

Responsive Web Design shifted us from fixed layouts and multiple sites to flexible systems that adapt to every screen. By applying fluid layouts, flexible media, and smart media queries, we build pages that look great and function well on any device.
